## Workflow для разработки бота

1. **Двигаться по итерациям из @tasklist.md строго сверху вниз**, без пропуска задач и итераций.
2. **Перед началом итерации**:
   - перечитать @vision.md и актуальные пункты @tasklist.md;
   - предложить краткий план итерации и ключевые изменения с примерными отрезками кода для согласования.
3. **После согласования плана**:
   - реализовать изменения минимально необходимым объёмом кода (принцип KISS);
   - не выходить за рамки согласованных файлов и объёма работ.
4. **После реализации**:
   - показать итоговые изменения с отрезками кода и кратким пояснением;
   - дождаться явного подтверждения на результат или корректировки.
5. **Обновление прогресса**:
   - отмечать выполненные пункты в @tasklist.md (чекбоксы и краткие комментарии при необходимости);
   - фиксировать, какая итерация сейчас завершена.
6. **Переход к следующей итерации**:
   - запросить согласие на старт следующей итерации;
   - при необходимости скорректировать порядок задач в пределах текущей итерации, не нарушая общий порядок.
7. **Работа с git**:
   - после завершения каждой итерации делать коммит с понятным сообщением;
   - один коммит — одна логически завершённая итерация.
