## Итерация 1 — самый простой скрипт

- [ ] Настроить окружение и зависимости (см. @README.md, @conventions.md).
- [ ] Реализовать минимальный CLI-запуск бота, который выводит фиксированный ответ на один входной промпт.
- [ ] Добавить простой лог запуска (в консоль или файл) для проверки, что бот реально отрабатывает.

## Итерация 2 — диалог в один сеанс

- [ ] Добавить поддержку многошагового диалога в рамках одного запуска (цикл ввода/вывода, команда выхода).
- [ ] Сохранить историю диалога в памяти процесса и выводить её по запросу (например, командой `/history`).
- [ ] Добавить простую обработку ошибок (сетевые/лимиты), чтобы бот не падал, а сообщал о проблеме.

## Итерация 3 — контекст проекта

- [ ] Научить бота читать файлы проекта по ссылкам вида `@path/to/file` и кратко их резюмировать.
- [ ] Ограничить объём читаемого текста (например, по размеру файла или числу строк), чтобы избежать перегрузки.
- [ ] Добавить команду для явной проверки связки с проектом (например, `/project-status`), которая показывает, что бот видит репозиторий.

## Итерация 4 — простые правки кода

- [ ] Реализовать команду, которая по запросу пользователя предлагает правку в указанном файле (через `@file`) и выводит diff.
- [ ] Добавить возможность применить или отклонить diff (например, командами `/apply` и `/discard` для последнего diff).
- [ ] Обновить логику так, чтобы после применения правки можно было сразу протестировать результат (запуск линтера/тестов по команде).

## Итерация 5 — рабочий цикл разработки

- [ ] Собрать единый сценарий «рабочего цикла»: пользователь даёт задачу → бот читает контекст (@doc/vision.md, @conventions.md и нужные файлы) → предлагает план → вносит изменения → подсказывает, как протестировать.
- [ ] Добавить короткий шаблон отчёта об итерации (что сделано, какие файлы тронуты, как воспроизвести тест).
- [ ] Обеспечить, чтобы после каждой итерации можно было обновлять блок прогресса сверху и делать короткую заметку об итогах.
